 Thank you for wanting to help pyspread. You can help me with bug reports, translations, icons, coding, ideas and much more.

contact
 For contributions, patches, development discussions and ideas please post to the pyspread-dev mailing list by sending an e-mail to pyspread-dev@gna.org. You can here subscribe to the list.

Adding your code
  1. Send an e-mail to pyspread-dev@gna.org that describes your contribution.
  2. Get a github account.
  3. Fork the branch that you would like to work with.
  4. Issue a pull request when ready.

code base

This informnation refers to pyspread v. 0.2.2.
Version 0.1.2 and previous have a different structure.

The code is organized in the src folder. There are the following sub-folders:

  • actions

Actions that change the model
  • gui

GUI elements including events
  • model

Core data structure
  • lib

Generically usable libraries

The figure on the right displays class instance dependencies of the different layers.

Relations to dependencies

wxPython related code can be found

  • in the action folder files,

  • in the gui folder files (including events.py),

  • in lib/clipboard.py,

  • in lib/gpg.py,

  • in lib/parsers.py,

  • in config.py,

  • in pyspread.py,

  • in sysvars.py and

  • in model/model.py.

The latter is necessary in order to use the wx constants in config.py


PyMe related code can be found in lib/gpg.py


numpy related code can be found in model/model.py
