How to help

Thank you for wanting to help pyspread. Bug reports, translations, icons, deployment to further operation systems, code, ideas etc. are most welcome.

For bug reports, please follow the link in the menu to the pyspread issue tracker on github. For anything else please send an e-mail to the author (see bottom of page).

Localization

Pyspread is localized via the Translation project, i.e. the textual content of the user interface is translated into different languages. If you like to localize pyspread, please refer to the translators' page for detailed instructions.

Full coverage of user interface (may still need polish):

  • Danish (da) by Joe Hansen
  • Ukrainian (uk) by Yuri Chornoivan

Partially translated user interface:

  • Brazilian Portuguese (pt_BR) by Enrico Nicoletto and Rafael Fontenelle
  • Dutch (nl) by Mark Haanen
  • French (fr) by Frédéric Marchal
  • German (de) by Mario Blättermann, Martin Manns, Christian Kirbach and Philipp Thomas
  • Nowegian Bokmaal (nb) by Andreas Noteng
  • Nowegian Nynorsk (nn) by Andreas Noteng

Details of the user interface localization status can be seen at the translation project page.

No translations of the user documentation (e.g. the First steps document ) are available, yet.

Interfaces

Only basic support for the Open Document Format (ODF) for exchanging spreaddheet data with other applications such as LibreOffice, Gnumeric etc. is available. It can be extended by adding features to the source file pyspread/src/interfaces/ods.py.

  3. Issue a pull request when ready.
